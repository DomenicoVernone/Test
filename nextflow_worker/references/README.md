# Legacy FreeSurfer-Based Processing Pipeline

These shell scripts are no longer directly used in the current workflow, but they provided the foundation for the automated pipeline defined in this project. They represent the original series of preprocessing and feature extraction steps applied to structural MRI data using **FreeSurfer** and related tools. For archival and reference purposes, they are stored in the `referenceses/shell` subfolder.

---

## Step 1: `freesurfer_1.sh` — Structural MRI Segmentation

This script performs **segmentation of structural MRI scans** using FreeSurfer for all subjects listed in the input .txt file.
For each subject, the script creates an output directory containing all segmentation results generated by FreeSurfer.

---

## Step 2: `create_nu.sh` — Conversion to NIfTI Format

This script takes the same .txt subject list as input and converts each subject’s `nu.mgz` file (a scalp-stripped, spatially normalized structural MRI image) into the **NIfTI format**.  
The converted files are stored within each subject’s respective directory created in Step 1.

> **Note:** The NIfTI format ensures spatial comparability across subjects and facilitates subsequent processing steps.

---

## Step 3: `create_ROI_from_freesurfer.sh` — ROI Extraction and Mask Generation

For each subject in .txt list and for each **Region of Interest (ROI)** defined in `data/all_freesurfer_labels.txt`, this script generates a NIfTI image corresponding to the segmented region.  
Within each subject’s folder, an `ROI` directory is created containing all regional brain images.

The extracted images are then **converted into binary masks** to be used in the following step.

---

## Step 4: `create_csv.sh` — CSV Construction for ROI Analysis

For each ROI defined in `all_freesurfer_labels.txt`, this script creates a `.csv` file structured as follows:

1. **Column 1:** Path to the subject’s `nu.nii` image  
2. **Column 2:** Path to the binary mask of the ROI for that subject  
3. **Column 3:** Subject’s age  
4. **Column 4:** Subject’s sex  

As a result, one `.csv` file is produced per region, containing metadata and file paths for all subjects included in the analysis.

---

## Step 5: `calc_feat_per_roi_norm.sh` — Radiomic Feature Extraction

For each region, this script reads the corresponding `.csv` file and computes **radiomic features** for every subject within that ROI.  
The output is a `.csv` file where each row represents one subject’s feature vector for the region under analysis.  
All output files are stored in the `feat` directory.

---

These scripts together established the conceptual and technical groundwork for the current automated MRI processing and feature extraction pipeline.


# Legacy Classification Pipeline

Alongside the FreeSurfer preprocessing workflow, an initial **training pipeline for classification** was developed using a set of R scripts.  
Although this early version laid the conceptual groundwork for the current machine learning framework, it has since been **greatly extended both in efficiency and in analytical scope**. The modern implementation now includes modular data handling, and generalized groups evaluation procedures.

---

## Script: `merge_radiomics_per_group_noshape.r`

This script merges all subject-level CSV files generated by the radiomic extraction process into a single dataset for group-wise analysis.  
The first line of the script specifies the path to the `features` directory containing the per-subject CSVs.

The output is a single combined file, `feat_all.csv`, saved in the same `features` directory.  
This file includes the radiomic features for each subject, grouped according to their diagnostic category.

> **Note:** The script was originally designed to compare only the groups **bvFTD** and **HC**.

---

## Script: `lasso.r`

This script takes as input the merged dataset `feat_all.csv` and performs **feature selection and classification** using several machine learning models.  
The script requires manual specification of two paths: the `features` directory (input) and the output folder for results (which must be created beforehand).

The script outputs five result files:
1. Four CSVs containing **classification performance metrics** for different models.
2. One CSV containing the **features selected** during each cross-validation fold using LASSO-based selection.

---

## Script: `rfe.r`

This script follows the same classification structure as `lasso.r` but introduces an additional **Recursive Feature Elimination (RFE)** step prior to model training.  
The goal of this approach is to iteratively identify the most informative subset of radiomic features, thereby improving model interpretability and reducing dimensionality before classification.

---

Together, these scripts established the **first complete version of the training pipeline**, which integrated radiomic feature aggregation, feature selection, and classification.  
The current system evolved from this foundation, incorporating advanced automation, reproducibility standards, and support for a wider variety of models and experimental designs.
